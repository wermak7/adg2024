Odczyt wartoci

Wartoci rastra mona wczyta na dwa sposoby -- definiujc punkt (QgsPointXY) lub blok (QgsRectangle). W drugim przypadku, jeli wskaemy wszystkie kolumny oraz wiersze, wtedy do pamici zostanie wczytany cay raster.

Punkt

  • jeli mamy wiele punktw i kilka kanaw, to oczywicie trzeba zastosowa ptl
  • ukad wsprzdnych rastra i punktw musi by jednakowy
  • alternatywnie mona uy identify()
  • jeli punkt ley poza obszarem, to zwrci nan

Docinanie rastra

Wczytany blok w rzeczywistoci jest niczym innym jak rastrem docitym przez zakres przestrzenny. Wynik tej operacji moemy zapisa na dysku.

from qgis.core import QgsRasterFileWriter, QgsRasterPipe, QgsCoordinateTransformContext

pipe = QgsRasterPipe()
provider = raster.dataProvider()
pipe.set(provider.clone())

writer = QgsRasterFileWriter("DEM_crop.tif")
writer.setCreateOptions(["COMPRESS=LZW"])
writer.setOutputFormat("GTiff")

status = writer.writeRaster(
    pipe,
    width,
    height,
    rect,
    raster.crs(),
    QgsCoordinateTransformContext()
)

if status != 0:
    print("Bd zapisu")
else: 
    print("OK")

Algebra rastrw

Klasa QgsRasterCalculator umoliwia dostp do kalkulatora rastrw w QGIS i wykorzystanie rnych operatorw uwzgldniajc matematyczne, trygonometryczne, porwnania czy logiczne (dziki czemu moliwe jest zamaskowanie czy reklasyfikacja wartoci rastra). Potencjalne zastosowania obejmuj:

  • uwypuklenie form rzeby terenu wykorzystujc wspczynnik przewyszenia,
  • wyznaczenie dolin rzecznych wykorzystujc maskowanie,
  • wyznaczenie klas terenu uywajc prostych regu klasyfikacyjnych.

Pamitaj, e w przypadku oblicze uwzgldniajcych kilka rastrw, to bezwzgldnie musz posiada jednakowe zasigi, rozdzielczo oraz ukady wsprzdnych.

from qgis.analysis import QgsRasterCalculator, QgsRasterCalculatorEntry

a = QgsRasterCalculatorEntry()
a.raster = raster
a.bandNumber = 1
a.ref = "DEM@1"

calculator = QgsRasterCalculator(
    "'DEM@1' * 3",                   # formulaString
    "output.tif",                    # outputFile
    "GTiff",                         # outputFormat
    raster.extent(),                 # outputExtent
    raster.crs(),                    # outputCrs
    raster.width(),                  # nOutputColumns
    raster.height(),                 # nOutputRows
    [a],                             # rasterEntries
    QgsCoordinateTransformContext()  # transformContext
)

status = calculator.processCalculation()

if status != 0:
    print("Bd zapisu")
else: 
    print("OK")

Maskowanie

Aby stworzy mask wartoci naley wykorzysta operator logiczny i ustalony warunek, np. wysoko terenu jest mniejsza ni 80 m n.p.m. W ten sposb otrzymamy raster binarny reprezentujcy obszary, ktre speniaj zadany warunek (warto 1; prawda) oraz nie speniaj (warto 0; fasz). Nastpnie, jeli przemnoymy mask przez wartoci rastra, to otrzymamy oryginalne wartoci dla obszarw speniajcych warunek, w przeciwnym razie iloczyn operacji wynosi 0 (czyli komra zostaje zamaskowana).

# maskowanie
formulaString = "('DEM@1' < 80) * 'DEM@1'"

Reklasyfikacja

W podobny sposb dziaa reklasyfikacja wartoci rastra. Jeli chcemy zreklasyfikowa raster na trzy klasy wysokoci, moemy zrobi to w nastpujcy sposb:

# reklasyfikacja
formulaString = "('DEM@1' < 94) * 1 + ('DEM@1' >= 94 AND 'DEM@1' <= 125) * 2 + ('DEM@1' > 125) * 3"

Przeprbkowanie

Do przeprbkowania (resampling) rastra mona wykorzysta dwa zaawansowane programy pochodzce z GDAL, tj. gdalwarp (uywany do transformacji rastrw midzy rnymi ukadami odniesienia) oraz gdal_translate (uywany do konwersji formatu i typw danych). Aby je zastosowa naley najpierw zainicjowa QGIS (bez tego nie bdzie moliwe znalezienie moduw przetwarzania).

# https://docs.qgis.org/latest/en/docs/user_manual/processing_algs/gdal/rasterprojections.html#warp-reproject
from qgis import processing

params = {
    "INPUT": filepath,          # nazwa obiektu lub cieka do pliku
    "TARGET_CRS": "EPSG:2180",  # reprojekcja
    "TARGET_RESOLUTION": 1000,  # rozdzielczo
    "RESAMPLING": 0,            # numer metody
    "OUTPUT": "DEM_1000.tif"
}

processing.run("gdal:warpreproject", params)

Alternatywnie mona uy odwoa systemowych poprzez modu Pythona subprocess do qgis_process.

import subprocess

params = [
    "--INPUT=" + filepath,
    "--TARGET_CRS=EPSG:2180",
    "--TARGET_RESOLUTION=1000",
    "--RESAMPLING=0",
    "--OUTPUT=DEM_1000.tif"
]
result = subprocess.run(["qgis_process", "run", "gdal:warpreproject"] + params,
                        capture_output = True, text = True)
print(result.stdout)

Tworzenie rastra

Tworzenie nowego rastra skada si z trzech zasadniczych krokw:

  1. Zainicjowanie pustego rastra na dysku (jeden lub wiele kanaw).
  2. Wczytanie wartoci i ich zastpienie.
  3. Aktualizacja (nadpisanie) wartoci rastra w pliku.

Po operacji zapisu, naley wyczy tryb edycji warstwy.

Na pocztku stwrzmy macierz z wartociami od 1 do 100, ktr zapiszemy na dysku.

czenie rastrw

Do czenia kilku jednokanaowych rastrw w jeden wielokanaowy mona wykorzysta podejcie zaprezentowane w poprzedniej sekcji dotyczcej tworzenia nowego rastra z t rnic, i zamiast metody createOneBandRaster naley uy createMultiBandRaster, w ktrej kady raster zostanie zapisany w ptli jako osobny kana. Pamitaj, e rastry musz posiada t sam liczb wierszy, kolumn, zakres oraz ukad przestrzenny. W przeciwnym razie wymagane jest ich wczeniejsze dopasowanie (przeprbkowanie).

Uproszczone podejcie opiera si o wykorzystanie dwch narzdzi GDAL:

  1. gdal_merge -- fizycznie czy dane z wielu rastrw w jeden plik.
  2. gdalbuildvrt -- tworzy wirtualny raster (plik z rozszerzeniem .vrt oparty o XML), ktry wycznie odwouje si do wejciowych plikw rastrowych bez ich kopiowania, tj. dziaa jako wskanik do oryginalnych plikw.
# https://docs.qgis.org/latest/en/docs/user_manual/processing_algs/gdal/rastermiscellaneous.html#merge

params = {
    "INPUT": [filepath, filepath],
    "SEPARATE": True,
    "NODATA_OUTPUT": 9999,
    "OUTPUT": "Merged.tif"
}

processing.run("gdal:merge", params)

# https://docs.qgis.org/latest/en/docs/user_manual/processing_algs/gdal/rastermiscellaneous.html#build-virtual-raster

params = {
    "INPUT": [filepath, filepath],
    "SEPARATE": True,
    "OUTPUT": "Merged.vrt"
}

processing.run("gdal:buildvirtualraster", params)
